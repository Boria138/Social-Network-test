# История обновлений Voxii

Все значимые изменения в проекте собраны в этом файле.

## [1.3.2] - 2026-03-17

### Добавлено
- Добавлен переключатель языка (`EN`/`RU`) на странице входа/регистрации.
- Добавлены переводы `auth.*` для экрана входа: заголовки, подписи полей, кнопки, валидация, ошибки и сообщения успеха.

### Изменено
- Изменили формат записи голосовых для лучшей совместимости: сначала используем `audio/mp4`/AAC (если браузер поддерживает), потом `audio/ogg`, и только в конце `audio/webm`.
- Для расшифровки (`/api/transcribe`) теперь отправляем файл с правильным расширением по его реальному MIME-типу, а не всегда как `.webm`.
- Переведены и подключены к i18n ранее хардкодные UI-строки в клиенте (friends/news/call/settings/voice/file preview).
- Для канала новостей добавлено корректное склонение счётчика подписчиков (`1 подписчик`, `2 подписчика`, `5 подписчиков`).
- Динамические элементы шапки и списка новостей теперь корректно обновляют язык при переключении.
- На мобильных устройствах нижняя панель серверов (`.server-list`) отключена, оставлен только выдвижной drawer каналов.

### Исправлено
- Убрали проблему, когда у голосовых иногда показывалась некорректная длительность (`Infinity`/`NaN`).
- Сделали запись стабильнее: перед остановкой принудительно забираем данные (`requestData()`), а запись ведем чанками (`start(250)`), чтобы реже появлялись пустые или битые файлы.
- Перед отправкой теперь проверяем, что голосовое не повреждено (размер файла и корректная длительность).
- Если запись все же битая, пробуем автоматически восстановить ее в WAV. Если восстановить нельзя, не отправляем файл и показываем понятную локализованную ошибку.
- Добавили переводы этой ошибки для `en` и `ru`.
- Исправлена ошибка формы входа `Недопустимый элемент управления формой ... username`: скрытые поля в режиме входа больше не остаются обязательными.
- Исправлено переполнение формы входа на экранах с ограниченной высотой (в т.ч. 16:10): добавлена адаптация по высоте и прокрутка.
- Исправлены кейсы, где отображались `Offline` и `Message yourself...` без локализации.
- Исправлено перекрытие поля ввода сообщений виртуальной клавиатурой на мобильных устройствах: добавлена адаптация через `visualViewport` и динамический отступ.

## [1.3.1] - 2026-03-16

### Добавлено
- Теперь можно закреплять сообщения в личных чатах (DM) и в чате с собой (Self Chat).
- Добавлена пересылка сообщений в другой чат через окно выбора получателя.
- Добавлены темы доступности: High Contrast, Deuteranopia Friendly и Tritanopia Friendly.
- Вместо emoji data URI теперь используется статический SVG favicon (`client/assets/favicon.svg`).
- Добавлены серверные реакции к новостям в системном канале (общие для всех пользователей).

### Изменено
- Приведены к единому стилю hover-эффекты кнопок действий сообщения (`reply`, `forward`, `edit`, `delete`, `reaction`).
- Эмодзи теперь везде рендерятся через Twemoji SVG, включая динамический контент.
- Символьные иконки интерфейса (`✓`, `✕`, `×`) заменены на встроенные SVG.
- Улучшена WebRTC-конфигурация для звонков между разными сетями и странами: расширен набор STUN/TURN и сетевые параметры подключения.
- Повышена стабильность mesh-конференций: добавлена защита от коллизий offer, фильтрация дублей ICE и более надежный ICE restart.
- Для аудио и видео sender-ов добавлены ограничения по битрейту и FPS, включая отдельный профиль для демонстрации экрана.
- Обновлен формат новости в системном канале: убран лишний `📢` из текстового контента и дублирование версии в заголовке.
- Для новостей добавлена загрузка реакций с сервера и синхронизация в реальном времени через Socket.IO.

### Исправлено
- Исправлена ошибка `socket is null` при ранней инициализации WebRTC-обработчиков.
- Регистрация WebRTC Socket.IO-обработчиков перенесена на этап после создания `socket`.
- Добавлены повторные попытки инициализации Socket.IO, если `io` загружается с задержкой.
- Добавлена защита от повторной регистрации WebRTC-обработчиков.
- Исправлена потеря форматирования при редактировании сообщений.
- Исправлено повторное редактирование: `data-raw-text` теперь корректно сохраняется и синхронизируется.
- Исправлено определение собственных сообщений в DM по `sender_id`.
- Восстановлены редактирование и удаление сообщений в Self Chat.
- Обновлены стиль, размер и иконка кнопки транскрибации голосовых сообщений.
- Исправлена обработка отложенных ICE-кандидатов после установки удаленного SDP у обеих сторон.
- Исправлено залипание `ignoreIncomingOffers` и улучшена дедупликация ICE-кандидатов для новых SDP/ICE-раундов.
- Убрано зеркалирование локального превью камеры в звонках.
- Исправлена передача звука при screen share: теперь можно отправлять микс `микрофон + звук вкладки/приложения`, а после остановки шаринга микрофон возвращается автоматически.
- Исправлен markdown-парсер новостей: теперь корректно поддерживаются вложенные списки (`-` с отступами).
- Убрано отображение пустого блока `message-text` у сообщений только с вложениями (голосовые, файлы), чтобы не появлялся пустой градиент.

## [1.3.0] - 2026-03-04

### Добавлено
- Добавлена транскрипция голосовых сообщений через whisper-cpp:
  - кнопка «Расшифровать» для голосовых сообщений;
  - автоматическое определение языка;
  - вставка результата в поле ввода;
  - поддержка многоязычной речи;
  - индикатор процесса транскрибации.

### Технические изменения
- Добавлен API-эндпоинт `/api/transcribe` для преобразования речи в текст.
- Добавлена интеграция whisper-cpp.
- Добавлена конвертация аудио через ffmpeg (16kHz, mono, 16-bit PCM).
- Добавлены переменные окружения для whisper-cpp:
  - `WHISPER_CPP_PATH` — путь к исполняемому файлу whisper-cpp;
  - `WHISPER_CPP_MODEL` — путь к модели whisper;
  - `WHISPER_CPP_ARGS` — дополнительные аргументы.
- Обновлен `deploy.sh` с автоматической установкой whisper-cpp.
- Добавлены i18n-переводы для кнопки транскрипции.
- Добавлена оптимизация для слабых VPS: модель tiny-q8_0 (~40MB RAM).
- Оптимизирован WebRTC: уменьшено количество STUN/TURN-серверов.
- Исправлена ошибка WebRTC `DOMException: The fetching process for the media resource was aborted`.

### Изменено
- Обновлена документация по SSL-конфигурации для WebRTC.
- Обновлена документация по установке whisper-cpp.

### Исправлено
- Исправлены записи в CHANGELOG.
- Исправлено дублирование video-элементов при WebRTC-звонках.
- Исправлена ошибка воспроизведения видео при повторных подключениях.

## [1.1.0] - 2026-02-24

### Добавлено
- Добавлена полноценная система уведомлений для сообщений и звонков:
  - браузерные push-уведомления (работают даже при закрытой вкладке);
  - звуковые уведомления для сообщений и входящих звонков;
  - счетчик непрочитанных сообщений на кнопке уведомлений;
  - бейджи непрочитанных на аватарах в списке контактов;
  - панель уведомлений с историей пропущенных звонков и сообщений;
  - хранение уведомлений в базе данных (синхронизация между устройствами);
  - индикаторы непрочитанных в списке DM.

### Технические изменения
- Добавлена таблица БД `notifications` для хранения уведомлений.
- Добавлены API-эндпоинты для уведомлений:
  - `GET /api/notifications` — получить все уведомления пользователя;
  - `GET /api/notifications/unread` — получить непрочитанные уведомления;
  - `POST /api/notifications/mark-all-read` — отметить все как прочитанные;
  - `POST /api/notifications/mark-user-read` — отметить уведомления от пользователя;
  - `DELETE /api/notifications/:id` — удалить уведомление;
  - `DELETE /api/notifications` — удалить все уведомления.
- Добавлен клиентский модуль `notification-service.js`.
- Обновлен интерфейс: добавлена кнопка уведомлений с бейджем.
- Сервер начал сохранять пропущенные звонки в базу данных.
- Добавлен рендер Twemoji для новостных сообщений.

### Изменено
- Уменьшен размер кнопок в панели пользователя, чтобы уместить кнопку уведомлений.

## [1.0.0] - 2026-02-23

### Добавлено
- Добавлен системный канал «Новости» для объявлений и обновлений платформы:
  - все пользователи подписываются автоматически;
  - канал создается автоматически при инициализации базы данных;
  - новые пользователи подписываются при регистрации.

### Технические изменения
- Добавлены таблицы БД: `channels`, `channel_subscriptions`, `channel_messages`.
- Добавлены API-эндпоинты для каналов:
  - `GET /api/channels` — получить все каналы пользователя;
  - `GET /api/channels/system` — получить системный канал;
  - `GET /api/channels/:channelId/messages` — получить сообщения канала;
  - `POST /api/channels/:channelId/messages` — отправить сообщение в канал.
- Добавлены Socket.IO-события для real-time обновлений каналов:
  - `send-channel-message` — отправка сообщения;
  - `new-channel-message` — получение нового сообщения.
- Обновлен клиентский интерфейс для отображения системного канала.
- Добавлены стили для выделения системного канала (оранжевый градиент).

## Формат журнала изменений

Проект использует формат [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

### Типы изменений
- **Добавлено** — новые функции.
- **Изменено** — изменения в существующем функционале.
- **Удалено** — удаленный функционал.
- **Исправлено** — исправления ошибок.
